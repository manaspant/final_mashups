What I did?

I made a game using the Twitter API. The user is supposed to play a game where he gets a few tweets from the twitter accounts of the most popular music celebrities (based on the number of followers). The user will also get the names of all the celebrities whose tweets I am using in a random order. In order to match the tweet to the celebrity, the player will have to click on the tweet or the name first, which will get selected. After that the player will have to click the other option which would depend on what was clicked first. If the answer is correct, both the options would flash green for 1.5 seconds and then dissapear. If the answer is incorrect then the two options will flash red and then they will go back to their original coor.

Why I did it?

Coming up with the idea of the project has always been the hardest part of the process for me. I spent a lot of time trying to look for ideas. I also changed my ideas again and again. Finally I decided to make a game about matching the tweets. A huge reason that I did this was that I wanted to do something challenging for my last project.

How I did it?

Initially I had no idea how I am supposed to start working on this as I had never made an online game before. So I started working with Craig's sample code to get tweets via the server. The first thing that I did was to modify the maketwittersearch function so that I could get the tweets from different users instead of just one. After I got that done I worked I worked on the logic of the game which involved setting up the mechanism for selecting different options and checking if the answer is correct. Next came the score keeping. After the game finished the user had to enter their name that would get stored in a databse and be displayed in the leaderboard.

Challenges faced

The first challenge I faced was in modyfing the twittersearch function as initially it could only get the tweets from a single user. Another challenge for me was making the user interface clear and simple for the user. I underestimated the time that I would need to improve the page layout and make the game user friendly. During the exhibition, I got good feedback watching the uders play the game, and I hope to incorporate some changes based on those observations.
